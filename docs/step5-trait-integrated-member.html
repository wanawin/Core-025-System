Step 5 — Trait-Integrated Member Engine

This step applies mined traits before weighted scoring. It is the first attempt to use hard trait overrides to improve Core 025 member selection.

Purpose

  • Use separation traits to force a member when evidence is strong
  • Use Top2-needed traits to force Top1+Top2 when needed
  • Use skip-danger traits to block bad skips
  • Fallback to weighted scoring when no strong override applies

Available Engine

  • v3.5 (Trait-Integrated)
    Uses mined traits as hard overrides before weighted member scoring
    View Python file

Required Inputs

  • Full history file
  • PLAY survivors file
  • Separation traits CSV
  • Top2-needed traits CSV
  • Skip-danger traits CSV

Optional inputs:

  • 0025 traits CSV
  • 0225 traits CSV
  • 0255 traits CSV

Override Priority

  1. Skip-danger traits block bad skips
  2. Top2-needed traits force Top1+Top2
  3. Separation traits can force a member
  4. Weighted scoring is used when no override applies

Current Status

  • This is the current trait-integrated member engine
  • It is designed to improve on the baseline member engine
  • It still requires historical walk-forward validation

⚠️ This step is experimental until it proves materially better capture in walk-forward testing.
