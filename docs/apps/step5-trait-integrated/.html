Step 5 — Trait-Integrated Member Engine

This step applies mined traits before weighted scoring. It is the first attempt to use hard trait overrides to improve member selection for Core 025.

Purpose

  • Use separation traits to force a member when evidence is strong
  • Use Top2-needed traits to force Top1+Top2 when required
  • Use skip-danger traits to block bad skips
  • Fallback to weighted scoring when no strong trait override applies

Available Engine

  • v3.5 (Trait-Integrated)

Optional inputs:

  • 0225 traits CSV

Current Status

  • This is the current trait-integrated member engine
  • It is designed to improve on the baseline member engine
  • It still requires full historical walk-forward validation

⚠️ This step is experimental and should not be considered validated until it proves materially better historical capture in walk-forward testing.
