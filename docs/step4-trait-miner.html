Step 4 — Member Trait Miner

This step mines historical seed-to-member transitions to find traits that may improve member selection for Core 025.

Purpose

  • Find separation traits where one member clearly beats the other two
  • Find member-specific traits for 0025, 0225, and 0255
  • Find Top2-needed pockets where Top1 often fails but Top2 succeeds
  • Find skip-danger pockets where skipping would throw away winners

Available Engine

  • Trait Miner v1
    Mines member-dominance, Top2-needed, and skip-danger traits
    View Python file

Expected Outputs

  • Separation traits CSV
  • 0025 traits CSV
  • 0225 traits CSV
  • 0255 traits CSV
  • Top2-needed traits CSV
  • Skip-danger traits CSV
  • Historical hit-event prediction table CSV

Current Status

  • The miner is functioning
  • Top2-needed and skip-danger outputs are especially important
  • At least one separation-traits run produced an effectively empty file

⚠️ An empty separation file is not just a file problem. It may mean the current settings are too strict or that strong single-trait member separation is limited.

How This Step Is Used

The outputs from this miner feed Step 5, where mined traits are applied as hard overrides before weighted member scoring.
